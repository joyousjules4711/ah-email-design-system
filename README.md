# Akademikerhilfe Email Design System

> Designed for Every Student Journey

The Akademikerhilfe Email Design System provides a consistent framework for designing, developing and maintaining transactional email templates across the organisation.

It combines reusable HTML components, documented design principles and standardized communication patterns into a single source of truth.

---

## Vision

Every email sent by Akademikerhilfe should provide the same user experience.

Whether a student receives

- a housing offer,
- a payment reminder,
- a SEPA notification,
- a move-in confirmation,
- or a password reset,

every communication should feel like it belongs to the same ecosystem.

---

## Goals

- Consistent corporate identity
- Reusable HTML components
- Responsive email templates
- Accessible layouts
- Simplified maintenance
- Multilingual support
- Centralised documentation

---

## Scope

The design system currently covers more than

- 150 master email templates
- German and English versions
- Department-specific templates
- Residence-specific variants

The long-term goal is to establish a single source of truth for all transactional email communication.

---

## Principles

The system is built around five principles.

### Consistent

Every email should look and feel like Akademikerhilfe.

### Human

Write for people, not for systems.

### Accessible

Every email should work on desktop, mobile devices and common email clients.

### Reusable

Components should be shared across all templates whenever possible.

### Maintainable

Updating one component should improve every email that uses it.
