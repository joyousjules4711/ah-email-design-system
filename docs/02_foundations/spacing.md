# Spacing

> **Applies to**
>
> - All transactional email templates
> - All reusable components
> - German and English versions

---

# Purpose

Spacing defines the rhythm of the Email Design System.

Consistent spacing improves readability, strengthens hierarchy and creates a calm visual appearance.

Spacing should be based on predefined design tokens rather than arbitrary values.

---

# Design Principles

Spacing should

- create hierarchy
- separate related information
- improve readability
- remain consistent

Whitespace is considered part of the design.

---

# Spacing Scale

| Token | Value |
|--------|------:|
| `space-2xs` | 4 px |
| `space-xs` | 8 px |
| `space-sm` | 12 px |
| `space-md` | 16 px |
| `space-lg` | 24 px |
| `space-xl` | 32 px |
| `space-2xl` | 48 px |
| `space-3xl` | 64 px |

These values should be reused throughout the design system.

---

# Component Spacing

## Header

Bottom spacing

`space-xl`

---

## Hero

Bottom spacing

`space-2xl`

---

## Section

Top

`space-xl`

Bottom

`space-xl`

---

## Headings

Spacing below headings

`space-md`

Avoid placing headings directly above other headings.

---

## Paragraphs

Spacing below paragraphs

`space-md`

Avoid excessive spacing between related paragraphs.

---

## Information Cards

Outer spacing

`space-xl`

Inner padding

`space-lg`

---

## Tables

Spacing above

`space-lg`

Spacing below

`space-xl`

Cell padding

`space-sm`

---

## Buttons

Spacing above

`space-lg`

Spacing below

`space-xl`

Buttons should never touch surrounding content.

---

## Signature

Spacing above

`space-2xl`

Spacing below

`space-xl`

---

## Footer

Top padding

`space-xl`

Bottom padding

`space-xl`

---

# Mobile Spacing

Desktop spacing may be reduced slightly on smaller screens.

Content should remain visually open and easy to scan.

Avoid removing whitespace entirely.

---

# Padding

Recommended content padding

Desktop

32 px

Mobile

24 px

---

# Margin Strategy

Prefer consistent spacing tokens over manually defined values.

Do not mix unrelated spacing values.

Example

Good

```
space-lg
space-xl
space-2xl
```

Avoid

```
19 px
27 px
41 px
```

---

# Best Practices

✔ Use spacing consistently.

✔ Prefer predefined spacing tokens.

✔ Let whitespace create hierarchy.

✔ Group related content.

✔ Separate unrelated content.

---

# Avoid

✘ Random spacing values.

✘ Crowded layouts.

✘ Inconsistent padding.

✘ Empty space without purpose.

---

# Guiding Principle

Spacing is an active design element.

It creates rhythm, improves readability and guides the recipient through the email.
